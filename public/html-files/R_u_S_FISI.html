1. Grundlagen der Netzwerktechnik (Network Basics)

Binär zu Dezimal (Binary to Decimal)

In der Netzwerktechnik ist es wichtig, zwischen Binär- und Dezimalzahlen umrechnen zu können:

  • 00000000 = 0
  • 11111111 = 255 = 1 + 2 + 4 + 8 + 16 + 32 + 64 + 128
  • 01000011 = 67 = 1 + 2 + 64

Dezimal zu Binär (Decimal to Binary)

  • 255 = 11111111 = 1 + 2 + 4 + 8 + 16 + 32 + 64 + 128
  • 38 = 00100110 = 2 + 4 + 32

Tipp: Jede Position in einer 8-Bit-Binärzahl hat einen Wert: 128, 64, 32, 16, 8, 4, 2, 1 (von links nach rechts).

IPv4-Adressen bestehen aus 32 Bit, aufgeteilt in 4 Oktette zu je 8 Bit:

  • Erste Adresse: 0.0.0.0
  • Letzte Adresse: 255.255.255.255
  • Beispiel: 192.168.2.1

IPv4 Address Format:

185 . 107 . 80 . 231
Oktet  Oktet  Oktet  Oktet

10110001.01101011.01010000.11100111
1 Byte = 8 Bits   1 Byte = 8 Bits   1 Byte = 8 Bits   1 Byte = 8 Bits
              
4 Bytes = 32 Bits

Subnetzmakse (Subnet Mask)

Die Subnetzmakse definiert, welcher Teil der IP-Adresse das Netzwerk und welcher Teil den Host identifiziert:

  • Standardsubnetzmakse für Klasse C: 255.255.255.0 (oder /24)
  • Bei dieser Maske sind die ersten 24 Bits für die Netzwerk-ID und die letzten 8 Bits für die Host-ID reserviert.

Tipp: Um festzustellen, ob zwei Geräte im selben Netzwerk sind, wenden Sie die Subnetzmakse auf beide IP-Adressen an - wenn das Ergebnis gleich ist, sind sie im selben Netzwerk.

2. Direkte Verbindung (Direct Connection)

Aufgabe 1.1.1: Direkte Verbindung

Verbindet zwei Laptops per LAN-Kabel (RJ45) miteinander und führt einen Ping zwischen den beiden Laptops durch.

  1. Besorgt euch zwei Laptops und ein RJ45-LAN-Kabel (Netzwerkkabel).
  2. Verbindet beide Laptops direkt mit dem Kabel.
  3. Konfiguriert die IP-Adressen auf beiden Laptops:
    • Windows: Netzwerkeinstellungen → Adapteroptionen ändern → Rechtsklick auf LAN-Verbindung → Eigenschaften → Internet Protocol Version 4 (TCP/IPv4) → Eigenschaften
    • Statische IP-Adressen im gleichen Netzwerk vergeben:
      Laptop 1: IP: 192.168.1.1, Subnetzmaske: 255.255.255.0
      Laptop 2: IP: 192.168.1.2, Subnetzmaske: 255.255.255.0
  4. Öffnet die Kommandozeile (cmd) und testet die Verbindung mittels Ping:
    # Auf Laptop 1: ping 192.168.1.2 # Auf Laptop 2: ping 192.168.1.1

Was ist ein "Ping"? Ein Ping ist ein Diagnose-Werkzeug, das ICMP Echo Request-Pakete an einen Zielhost sendet und auf Antworten wartet. Es wird verwendet, um die Erreichbarkeit eines Hosts zu testen und die Übertragungszeit zu messen.

Achtung: Falls der Ping nicht funktioniert, überprüft folgende Punkte:

  • Sind die IP-Adressen korrekt konfiguriert?
  • Ist die Windows-Firewall eventuell ein Hindernis? (ICMP könnte blockiert sein)
  • Funktioniert das Netzwerkkabel? (Leuchten die LAN-LEDs?)

3. Indirekte Verbindung (Indirect Connection)

Aufgabe 1.1.2: Indirekte Verbindung

Schafft nun aus mehreren Laptops ein LAN. Bei diesem sollen alle Geräte miteinander kommunizieren können.

  1. Besorgt euch einen Netzwerk-Switch als zwischengeschaltetes Gerät.
  2. Verbindet alle Laptops mit LAN-Kabeln mit dem Switch.
  3. Konfiguriert IP-Adressen auf allen Laptops im gleichen Netzwerk:
    • Laptop 1: 192.168.1.1/24
    • Laptop 2: 192.168.1.2/24
    • Laptop 3: 192.168.1.3/24
    • usw.
  4. Testet die Verbindung zwischen allen Geräten mit dem Ping-Befehl.

Was ist ein Switch? Ein Switch ist ein Netzwerkgerät, das mehrere Geräte in einem lokalen Netzwerk (LAN) verbindet. Im Gegensatz zu einem Hub leitet ein Switch Pakete nur an den spezifischen Port weiter, an dem das Zielgerät angeschlossen ist, was die Netzwerkeffizienz verbessert.

Indirekte Verbindung mit Switch:

Laptop 1 (192.168.1.1) -----+
                            |
Laptop 2 (192.168.1.2) -----+------ Switch -----+
                            |                    |
Laptop 3 (192.168.1.3) -----+                    +------ Optional: Router zum Internet
                                                 |
                                                 +
                                          Weitere Geräte

4. Routing zwischen Netzwerken (Routing Between Networks)

Aufgabe 1.1.3: Routing

Verbindet dieses LAN mit einem zweiten, neuen LAN. Dieses Netzwerk soll unabhängig vom anderen sein. Findet eine Möglichkeit, diese beiden Netzwerke miteinander kommunizieren zu lassen.

  1. Erstellt ein zweites Netzwerk mit einem anderen IP-Bereich:
    • Netzwerk 1: 192.168.1.0/24
    • Netzwerk 2: 192.168.2.0/24
  2. Verbindet beide Netze zunächst mit einem Switch:
    • Konfiguriert Geräte im Netzwerk 1 mit IP-Adressen aus dem 192.168.1.0/24 Bereich
    • Konfiguriert Geräte im Netzwerk 2 mit IP-Adressen aus dem 192.168.2.0/24 Bereich
  3. Prüft: Können sich die Laptops der unterschiedlichen Netze jetzt anpingen? (Normalerweise nicht, da sie in verschiedenen Subnetzen sind)
  4. Router als Vermittler einrichten:
    • Verbindet den Router mit beiden Netzwerken
    • Konfiguriert den Router mit je einer IP-Adresse in beiden Netzwerken:
      Interface 1: 192.168.1.254/24
      Interface 2: 192.168.2.254/24
    • Aktiviert IP-Routing auf dem Router (siehe Router-Konfigurationsabschnitt)
  5. Konfiguriert auf allen Geräten das Default-Gateway:
    • Geräte in Netzwerk 1: Gateway = 192.168.1.254
    • Geräte in Netzwerk 2: Gateway = 192.168.2.254
  6. Testet die Verbindung zwischen den Netzwerken mit Ping.

Was ist ein Router? Ein Router ist ein Netzwerkgerät, das Datenpakete zwischen verschiedenen Netzwerken weiterleitet. Er arbeitet auf Schicht 3 (Netzwerkschicht) des OSI-Modells und verwendet IP-Adressen zur Weiterleitung.

Routing zwischen zwei Netzwerken:

Netzwerk 1 (192.168.1.0/24)                      Netzwerk 2 (192.168.2.0/24)
+-------------------+                           +-------------------+
|                   |                           |                   |
| Laptop 1.1        |                           | Laptop 2.1        |
| 192.168.1.1/24    |                           | 192.168.2.1/24    |
| Gateway: 1.254    |                           | Gateway: 2.254    |
|                   |                           |                   |
+--------+----------+                           +---------+---------+
         |                                                |
+--------+----------+                           +---------+---------+
|                   |                           |                   |
| Laptop 1.2        |                           | Laptop 2.2        |
| 192.168.1.2/24    |                           | 192.168.2.2/24    |
| Gateway: 1.254    |                           | Gateway: 2.254    |
|                   |                           |                   |
+--------+----------+                           +---------+---------+
         |                                                |
         |                                                |
         |          +---------------------+               |
         +----------| Router              |---------------+
                    | Interface 1: 1.254  |
                    | Interface 2: 2.254  |
                    +---------------------+

5. Knowledge Proof (Wissensüberprüfung)

Aufgabe 1.2: Knowledge Proof

Frage: Was ist die letztmögliche IP-Adresse in einem Netzwerk mit der Subnetzmaske 255.255.255.0 und der Netzadresse 10.10.10.0? Was ist die erstmögliche?

Antwort:

  • Netzadresse (Network Address): 10.10.10.0
  • Subnetzmaske (Subnet Mask): 255.255.255.0 (/24)
  • Erstmögliche Host-Adresse: 10.10.10.1
  • Letztmögliche Host-Adresse: 10.10.10.254
  • Broadcast-Adresse: 10.10.10.255

Erklärung: Bei einer /24 Subnetzmaske sind die ersten 24 Bits (die ersten drei Oktette) für die Netzwerk-ID reserviert, und die letzten 8 Bits für die Host-ID. Die Netzadresse hat alle Host-Bits auf 0, die Broadcast-Adresse hat alle Host-Bits auf 1. Die nutzbaren Host-Adressen liegen dazwischen.

Frage: Wie viele Hosts lassen sich in diesem Netzwerk gleichzeitig einbinden und warum?

Antwort: In einem Netzwerk mit einer /24 Subnetzmaske (255.255.255.0) können 254 Hosts eingebunden werden.

Erklärung: Bei 8 Bit für Host-Adressen gibt es theoretisch 2^8 = 256 mögliche Kombinationen. Davon müssen jedoch zwei abgezogen werden:

  • Die Netzadresse (alle Host-Bits sind 0)
  • Die Broadcast-Adresse (alle Host-Bits sind 1)
Daher: 256 - 2 = 254 nutzbare Host-Adressen

Frage: Was ist im Regelfall das Standardgateway im Netz 192.168.2.0/24?

Antwort: Im Regelfall wird oft die erste (.1) oder letzte nutzbare Adresse (.254) als Standardgateway verwendet. In einem 192.168.2.0/24 Netzwerk wäre das typischerweise entweder 192.168.2.1 oder 192.168.2.254.

Erklärung: Es gibt keine technische Regel, welche IP-Adresse als Gateway genutzt werden muss. Allerdings ist es eine weit verbreitete Konvention, entweder die erste (192.168.2.1) oder die letzte nutzbare Adresse (192.168.2.254) im Subnetz für das Gateway zu verwenden.

Frage: Was passiert, wenn zwei Laptops die gleiche IP-Adresse haben? Probiert es aus!

Antwort: Wenn zwei Geräte in einem Netzwerk die gleiche IP-Adresse verwenden, kommt es zu einem IP-Adresskonflikt. Dies führt zu unvorhersehbarem Verhalten der Netzwerkkommunikation für beide Geräte.

Erklärung: Typische Symptome:

  • Beide Geräte erhalten Fehlermeldungen über einen IP-Konflikt
  • Intermittierende Verbindungsverluste
  • Pakete werden an das falsche Gerät gesendet
  • Unzuverlässige Netzwerkverbindung für beide Geräte

In Windows erscheint normalerweise eine Benachrichtigung, die auf einen IP-Konflikt hinweist.

Frage: Wie wirkt sich eine Anpassung der Subnetzmaske an einem der PCs, ohne Änderung an den anderen Netzwerkteilnehmern, auf die Funktion des Netzwerkes aus?

Antwort: Wenn die Subnetzmaske nur auf einem PC geändert wird, kann dies dazu führen, dass dieser PC bestimmte andere Geräte nicht mehr erreichen kann, obwohl physisch alle im selben Netzwerk verbunden sind.

Erklärung:

  • Eine PC bestimmt anhand seiner IP-Adresse und Subnetzmaske, ob ein Zielgerät im gleichen Subnetz liegt
  • Wenn die Subnetzmaske nur auf einem PC geändert wird, ändert sich dessen Sicht des Netzwerks
  • Beispiel: Mit 192.168.1.10/24 wird ein PC mit 192.168.1.20 als im lokalen Netz angesehen. Mit 192.168.1.10/16 würde auch 192.168.2.20 als im lokalen Netz angesehen werden.
  • Dies kann zu Routing-Problemen führen, da der PC möglicherweise versucht, manche Pakete direkt zu senden, statt sie über das Gateway zu routen, oder umgekehrt.

6. Weiterführende Aufgaben (Advanced Tasks)

Aufgabe 1.3: Weiterführende Aufgaben

Frage: Was ist eine Routing-Tabelle und wie erstellt man Einträge in dieser?

Antwort: Eine Routing-Tabelle ist eine Datenstruktur, die von einem Router oder einem Gerät mit Routing-Funktionen verwendet wird, um zu bestimmen, wohin Netzwerkpakete weitergeleitet werden sollen.

Windows Routing-Tabelle anzeigen:

route print

Statische Route in Windows hinzufügen:

route ADD Zielnetzwerk MASK Subnetzmaske Gateway-IP

Beispiel:

route ADD 192.168.3.0 MASK 255.255.255.0 192.168.1.254

Cisco Router Routing-Tabelle anzeigen:

show ip route

Statische Route auf Cisco Router hinzufügen:

Router# config terminal Router(config)# ip route Zielnetzwerk Subnetzmaske Next-Hop-IP Router(config)# exit Router# write mem

Beispiel:

Router# config terminal Router(config)# ip route 192.168.3.0 255.255.255.0 192.168.2.1 Router(config)# exit Router# write mem

Frage: Welche Dienste werden für einen Ping benötigt?

Antwort: Ping verwendet das Internet Control Message Protocol (ICMP), insbesondere die ICMP Echo Request und Echo Reply Nachrichten.

Erklärung:

  • ICMP ist ein Protokoll der Netzwerkschicht (Layer 3) im OSI-Modell
  • Ping sendet ein ICMP Echo Request (Typ 8) an das Zielgerät
  • Das Zielgerät antwortet mit einem ICMP Echo Reply (Typ 0)
  • Firewalls und Router müssen ICMP-Verkehr erlauben, damit Ping funktioniert
  • Ping benötigt keine speziellen Dienste auf Anwendungsebene, wie einen Server oder Daemon

Hinweis: In vielen Netzwerken werden ICMP-Nachrichten aus Sicherheitsgründen gefiltert oder blockiert, besonders an Netzwerkgrenzen.

Frage: Wenn du an deinem Handy eine Website aufrufen möchtest, musst du dafür keine IP im Browser aufrufen, sondern gibst nur eine URL ein. Woher weiß der Browser welche IP gefragt ist?

Antwort: Der Browser verwendet das Domain Name System (DNS), um den Domainnamen in eine IP-Adresse aufzulösen.

Der DNS-Auflösungsprozess:

  1. Du gibst eine URL wie "www.example.com" ein
  2. Der Browser prüft seinen lokalen Cache nach der IP-Adresse
  3. Falls nicht gefunden, fragt das Gerät beim konfigurierten DNS-Server nach
  4. Der DNS-Server löst den Namen auf und gibt die IP-Adresse zurück
  5. Der Browser stellt nun eine Verbindung zur erhaltenen IP-Adresse her

Frage: Wie nennt sich die Technik/das Protokoll die dabei eingesetzt wird?

Antwort: Die Technik nennt sich "Domain Name System" (DNS). Das Protokoll verwendet typischerweise UDP auf Port 53 für einfache Abfragen und kann auf TCP für größere Antworten oder Zonentransfers zurückgreifen.

7. Router-Konfiguration (Router Configuration)

Router zurücksetzen und konfigurieren

  1. Router ausschalten
  2. Den Router via COM-Port zu RJ-45 am "Console" Steckerplatz verbinden
    Einstellungen: Baud: 9600, Data-Bits: 8, Stop-Bits: 1, Parity: none, Flow-Control: none
  3. Mit Putty oder anderem Terminalprogramm auf COM1 verbinden
  4. Router anschalten und in den ersten 15 Sekunden STRG+Pause drücken
  5. Folgende Commands eingeben:
    confreg 0x2141 reset
  6. Nach Neustart des Routers bei der Abfrage, ob eine Initialkonfiguration durchgeführt werden soll, "no" eingeben
  7. Enter drücken
  8. Danach folgende Commands eingeben:
    en write erase y configure t config-register 0x2102 end
  9. Den Router neustarten
  1. Would you like to enter the initial configuration dialog? [yes/no]: yes
  2. Would you like to enter basic management setup? [yes/no]: yes
  3. Enter host name [Router]: <dein Hostname>
  4. Enter enable secret: <Dein Secret Passwort>
  5. Enter enable password: <Dein Passwort>
  6. Enter virtual terminal password: <Dein Virtual Terminal Passwort>
  7. Configure SNMP Network Management? [yes]: no
  8. Enter interface name used to connect to the management network from the above interface summary: GigabitEthernetX/X (Kommt drauf an was für Interfaces der Router hat)
  9. Use the 100 Base-TX (RJ-45) connector? [yes]: yes
  10. Operate in full-duplex mode? [no]: auto
  11. Configure IP on this interface? [yes]: yes
  12. IP address for this interface: <IP-Adresse>
  13. Subnet mask for this interface [255.255.0.0]: <Subnetzmaske>
  14. [0] Go to the IOS command prompt without saving this config.
    [1] Return back to the setup without saving this config.
    [2] Save this configuration to nvram and exit.
    Enter your selection [2]: 2

Privilegierte Rechte bekommen:

Router> enable Password:

Interface Konfiguration:

Router# config terminal Router(config)# interface GigabitEthernetX/X Router(config-if)# ip address Router(config-if)# no shutdown Router(config-if)# exit Router# write mem

Das Kommando no shutdown bewirkt, dass das Interface bei einem Neustart direkt angeschaltet wird.

IP-Routing aktivieren:

Router# config terminal Router(config)# ip routing Router(config)# exit Router# write mem

Statische Routen hinzufügen:

Router# config terminal Router(config)# ip route Router(config)# exit Router# write mem

Beispiel:

Router# config terminal Router(config)# ip route 192.168.2.0 255.255.255.0 192.168.1.2 Router(config)# exit Router# write mem

Nützliche Befehle zur Überprüfung:

Router# show ip route Router# show running-config interface GigabitEthernetX/X

8. Nützliche Befehle (Useful Commands)

  • ipconfig /all - Zeigt detaillierte Netzwerkkonfiguration an
  • ping [IP-Adresse] - Testet die Verbindung zu einem Gerät
  • tracert [IP-Adresse] - Zeigt den Weg eines Pakets zum Ziel
  • netstat -an - Zeigt alle aktiven Netzwerkverbindungen
  • nslookup [Domainname] - Löst einen Domainnamen in eine IP-Adresse auf
  • route print - Zeigt die Routing-Tabelle an
  • route add [Netzwerkadresse] mask [Subnetzmaske] [Gateway] - Fügt eine statische Route hinzu
  • arp -a - Zeigt die ARP-Tabelle (IP zu MAC-Adresse Zuordnung) an

Basis-Befehle:

  • show running-config - Zeigt die aktuelle Konfiguration
  • show ip interface brief - Übersicht aller Interfaces mit IP-Adressen
  • show ip route - Zeigt die Routing-Tabelle
  • ping [IP-Adresse] - Testet die Verbindung zu einem Gerät
  • traceroute [IP-Adresse] - Zeigt den Weg eines Pakets zum Ziel

Konfigurationsbefehle:

  • enable - Wechselt in den privilegierten Modus
  • configure terminal - Wechselt in den Konfigurationsmodus
  • interface [Interface-Name] - Konfiguriert ein bestimmtes Interface
  • ip address [IP-Adresse] [Subnetzmaske] - Setzt die IP-Adresse eines Interfaces
  • no shutdown - Aktiviert ein Interface
  • ip route [Zielnetzwerk] [Subnetzmaske] [Nächster Hop] - Fügt eine statische Route hinzu
  • write memory oder copy running-config startup-config - Speichert die Konfiguration
